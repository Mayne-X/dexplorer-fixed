### Task 2: Light-Mode Amber Contrast

**Files:**

- Modify: `src/theme/colors.ts` lines 5–6
- Modify: `src/index.css` lines 77–81

The current light primary `#D97706` (Amber-600) gives ~3:1 on white — fails WCAG AA (4.5:1 required for normal text). Fix: Amber-700 `#B45309` gives ~4.6:1.

- [ ] **Step 1: Update light.primary and light.primaryHover in colors.ts**

In `src/theme/colors.ts`, change lines 5–7:

```ts
    primary: '#B45309',      // Amber-700 — 4.6:1 on white (WCAG AA)
    primaryHover: '#92400E', // Amber-800
```

- [ ] **Step 2: Update static CSS vars in index.css to match**

In `src/index.css`, change the `:root` block (lines 77–81):

```css
:root {
  --primary-color-20: rgba(180, 83, 9, 0.2);
  --accent-color: #7c3aed;
  --primary-color: #b45309;
}

.dark {
  --primary-color-20: rgba(245, 158, 11, 0.2);
  --accent-color: #8b5cf6;
  --primary-color: #f59e0b;
}
```

- [ ] **Step 3: Run lint/check**

```
pnpm lint && pnpm check
```

- [ ] **Step 4: Commit**

```

```

- [ ] **Step 1: Replace nav Link hover handlers**

```tsx

                      onClick={handleMobileLinkClick}
                      className={cn(
                        'group flex gap-x-3 rounded-lg px-3 py-2.5 text-sm font-medium transition-all duration-200 relative font-ui',
                        isActive
                          ? ''
                          : 'hover:bg-[var(--color-background-secondary)] hover:text-[var(--color-text-primary)] hover:translate-x-1'

                          : colors.text.secondary,
```
